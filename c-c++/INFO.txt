constexpr in C++ is a keyword that indicates that the value of a variable or the result of a function can be evaluated at compile time.

In short:

    Use constexpr for variables or functions that can be computed during compilation, which can improve performance.

    constexpr guarantees compile-time constantness, unlike const which just means immutable after initialization but might still be computed at runtime.

Example with variable:

constexpr int square(int x) {
    return x * x;
}

constexpr int val = square(5);  // computed at compile time

Example with function:

constexpr int factorial(int n) {
    return n <= 1 ? 1 : (n * factorial(n - 1));
}

constexpr int f5 = factorial(5); // evaluated at compile time

1. static_cast<T>(expr)

    Safe, compile-time checked conversions.

    Used for well-defined conversions:

        numeric conversions (e.g., int → float)

        pointer upcasts (derived → base)

        enum ↔ integer conversions

    Does not remove const or perform unsafe casts.

Example:

int i = 10;
double d = static_cast<double>(i);  // int → double

2. dynamic_cast<T>(expr)

    Used mainly for downcasting pointers or references in polymorphic hierarchies (classes with virtual functions).

    Performs runtime type check and returns nullptr if cast fails (for pointers).

    Requires RTTI enabled.

Example:

Base* basePtr = new Derived();
Derived* derivedPtr = dynamic_cast<Derived*>(basePtr);
if (derivedPtr) {
    // safe to use derivedPtr
}

3. const_cast<T>(expr)

    Removes or adds const or volatile qualifiers.

    Only used to cast away constness or volatility.

    Unsafe if you modify something originally declared const.

Example:

const int x = 5;
int* ptr = const_cast<int*>(&x);
*ptr = 10;  // Undefined behavior!

4. reinterpret_cast<T>(expr)

    Performs low-level, unsafe conversions.

    Converts any pointer to any other pointer type (e.g., between unrelated classes, between pointer and integer).

    Rarely portable or safe.

    Use carefully.

Example:

int i = 65;
char* p = reinterpret_cast<char*>(&i);

A union in C++ is a special user-defined type similar to a struct, but with one critical difference:

    🚨 All members of a union share the same memory location.

This means:

    A union can only hold one value at a time.

    The size of a union is equal to the size of its largest member.

    You can use it to reinterpret data without copying.

✅ Syntax Example:

union MyUnion {
    int i;
    float f;
};

You can use it like this:

MyUnion u;
u.i = 42;
std::cout << u.i << "\n";  // OK
u.f = 3.14f;                // This overwrites the int
std::cout << u.f << "\n";  // OK

But now u.i holds garbage, because u.f overwrote its memory.
📦 In Practice – Why We Use It for Type Casting:

Here’s your float example:

union {
    float f;
    uint32_t u;
} conv;

conv.f = 3.14f;
std::cout << std::hex << conv.u << "\n";  // Bit-level representation of the float

| `\W`      | Non-word character                  | `!`, `.`               |            |
| `\s`      | Whitespace (space, tab, newline)    | `' '`                  |            |
| `\S`      | Non-whitespace                      | `a`, `3`               |            |
| `^`       | Start of string                     |                        |            |
| `$`       | End of string                       |                        |            |
| `[...]`   | Character class                     | `[abc]` matches `a`    |            |
| `[^...]`  | Negated character class             | `[^a]` not `a`         |            |
| \`a       | b\`                                 | Alternation            | `a` or `b` |
| `( ... )` | Capturing group                     |                        |            |
| `*`       | 0 or more                           | `lo*` → `l`, `loo`     |            |
| `+`       | 1 or more                           | `lo+` → `lo`, `loo`    |            |
